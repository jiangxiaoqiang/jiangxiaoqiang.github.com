如何高效工作

发表于 2017-03-12 | 分类于 Working |

高效率的工作离不开外部环境和自身环境。本文就自己工作中遇到的一些影响工作效率的痛点与改进建议与大家分享。其中没有重要与不重要一说，每个都非常重要，各种因素互相协同，如果都做到足够优秀，工作效率自然而然就上去了。如果有任何一项有短板，都需要大家一起努力想办法进行改进。

高速无障碍的网络

现在社会信息量这么大，我们不可能记住所有的东西，就拿编程语言来说，常用的语言就有几十上百种，还有其他无法精确统计的小众语言。每种语言有对应的一套技术栈与工具集。是没法所有的都记住的，但是有搜索引擎，有时只需要知道某部分就可以了，其他的通过资料搜索即可找到。所以网络对于工作来说是非常重要的，高速网络就像空气，平时你感觉不到，等到你用上低速网络时，才会意识到高速网络是多么可贵。还有非常重要的一点是网络必须是足够开放的 ，因为越是开放的网络，找到与你遇到相同问题的人解决问题的思路的概率越大。你要相信自己不是地球上第一个遇到这种问题的人，好多问题前人已经有现成的解决方案(Best Practice)，你不用重新造轮子，需要做的就是找到他。最近使用的网络10MB带宽几十个人使用，网页都无法打开。如果需要安装个软件或者查点资料都无法办到。还好换了蚂蚁宝卡，直接将4G网络连接共享给PC，10块钱就可以购买1GB流量，至少查资料的问题可以完美解决了。

高配的设备

设备对于工作，就像武士的刀一样，垃圾的武士刀切切菜都不够顺手。最低要求是不能卡，不仅能卡端节奏，还能卡断思路。

高效的沟通方式

工作中经常遇到的问题就是，客户提出了一个需求，需要与不同的人反复确认，反复沟通。而且每个人说的需求都有差别甚至是相反的。如何统一需求的口径？需求如何无缝的传达给每个人？即使他加入团队词才不到一周，也可以快速的明白整个系统架构与设计，这个地方为什么要这样设计？这个按钮背后的逻辑是怎样的？这里是什么流程？为什么要这么制定流程？为什么这里和那里有那么一点点区别？

合理的分工

有时会发现，要完成一件事至少需要多个人的配合。要做事情A必须要B的配合，这样无形也极大的降低了效率。所以在分工时，尽量保证一个人不用其他人配合就可以完成整个流程。

良好的团队氛围

个人以为良好的团队氛围是可以无压力高效沟通，其实软件开发已经不会涉及倒多大的技术难度，最大的问题就是沟通，将心中的想法或客户的想法变成软件。开发出来的东西是不是客户（用户）需要的？所以沟通可以说是起到的决定作用。

如果没有以上任何一个痛点，应该是极好的。
